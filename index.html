ZTP — Zyquo Tool Protocol

A native macOS agent tool runtime, written in Swift. ZTP gives an AI agent twelve battle-tested, schema-validated tools — Office document generation, charting, email & messaging, headless browsing, macOS automation, local OCR, Apple Notes, file management, and Finder control — behind one consistent protocol.

macOS 14+ Swift 6 · strict concurrency arm64 12 tools 278 tests passing notarized · Developer ID Apache-2.0

Overview #

ZTP is the execution layer behind the Zyquo AI agent, but it ships as a standalone CLI you can use directly. Every tool conforms to one protocol — a typed manifest plus an execute(input, context) method — so the agent (or you) can discover, schema-validate, and invoke them uniformly.

🧱 Native & fast

Pure Swift. OpenXML/OOXML written from scratch — no Python, no Node, no LibreOffice.

🔐 Safe by default

Mutating & destructive ops are --confirm-gated; AppleScript is escaped and runs without a shell; the browser blocks SSRF hosts.

🤝 Two interfaces

Friendly per-tool subcommands and a generic protocol (run/tools/schema/inspect/validate) for agents.

📦 Self-contained

One signed, notarized universal binary. Optional system tools detected at runtime.

Quick start #

# Generate a spreadsheet from a JSON spec
ztp excel build report.json --output report.xlsx --json

# Extract text from a scanned PDF (local OCR, no network)
ztp ocr pdf invoice.pdf --pages 1-2 --output invoice.txt

# Search a tree by content, then zip the matches' folder
ztp files search --path ~/src --query "TODO" --content --ext swift --json
ztp files compress --path ~/src --output ~/src.zip

# Drive Finder
ztp finder set-view --view column --path ~/Downloads

Protocol commands #

Beyond the per-tool subcommands, ZTP exposes a generic protocol surface that agents use to discover and call any tool by name:

CommandPurpose
ztp tools --jsonList every built-in tool with its manifest.
ztp schema <tool>Emit the JSON-schema-like input description (commands + params).
ztp inspect <tool>Human-readable capabilities & command reference.
ztp run <tool> <input.json>Invoke a tool with a flat JSON object (command selects the op).
ztp validate <tool> <input.json>Validate an input payload against the tool schema.
Every tool returns a structured result: { ok, tool, duration_ms, data | error }. Always check ok before chaining.

The 12 tools #

Document & visual generators, communications, web, and system automation.

📊 ztp-excel

XLSX · OpenXML from scratch
build · validate-spec · validate · inspect · import-csv

Generate fully-featured XLSX workbooks from JSON: multiple sheets, formulas, number formats & styles, merged cells, column widths, frozen panes, data validation (dropdowns / numeric ranges), conditional formatting (data bars & color scales), and cell comments.

ztp excel build report.json --output report.xlsx --json
ztp excel import-csv data.csv --output data.xlsx --infer-types
ztp excel inspect report.xlsx --json

📝 ztp-docx

DOCX · OpenXML from scratch
build · validate-spec · validate · inspect

Word documents from JSON: headings, styled runs, paragraph alignment, tables with column widths, images, page breaks, horizontal rules, headers/footers (with page numbers), hyperlinks, and nested lists (levels 0–8).

ztp docx build proposal.json --output proposal.docx --json

📑 ztp-slides

PPTX · OpenXML from scratch
build · validate-spec · validate · inspect

PowerPoint decks from JSON: titled slides, bullets, images, speaker notes (notesMaster + notesSlide), slide numbers, and per-slide transitions.

ztp slides build deck.json --output deck.pptx --json

📈 ztp-chart

PNG · SVG · PDF
build · validate-spec · inspect · data-summary · themes

Render bar, line, scatter, pie, area, and histogram charts from data specs. Supports axis min/max, log scales, stacked bars, data labels, and built-in themes.

ztp chart build sales.json --output sales.svg --format svg
ztp chart themes

✉️ ztp-mail

SMTP · Apple Mail · RFC 5322
validate · preview · draft · send · apple-draft · inspect

Draft, preview, and send email. Markdown/HTML bodies, attachments, priority headers, custom headers (CRLF-injection-guarded), and inline images (multipart/related + Content-ID). send requires --confirmed.

ztp mail preview message.json --output preview.html
ztp mail draft message.json --output message.eml
ztp mail send message.json --profile work --confirmed

💬 ztp-message

iMessage · SMS · AppleScript
validate · preview · draft · send · apple-draft · templates · inspect

Draft and send iMessage/SMS via Messages.app. Validates recipients, warns on SMS segmentation, and ships message templates. send requires --confirmed; AppleScript is fully escaped.

ztp message preview msg.json
ztp message send msg.json --confirmed

🌐 ztp-browser

WebKit headless · SSRF-guarded
screenshot · pdf · html · text · links · metadata · inspect · open · run · safari-*

Headless web automation: full-page screenshots, PDF export, raw HTML, readable text, link & metadata extraction. Custom HTTP headers, viewport control, per-strategy wait settle. Private/loopback/metadata hosts are blocked by default (SSRF).

The url is a positional argument: ztp browser text <url>.

ztp browser text https://example.com --json
ztp browser screenshot https://example.com --output shot.png --width 1440
ztp browser pdf https://example.com --output page.pdf

🖥️ ztp-macos

AppleScript · Process · Files · --confirm gating
system-* · finder-* · files-* · clipboard-* · notify · screenshot-* · apps-* · windows-list · processes-* · applescript-* · shortcuts-* · permissions-check

Broad macOS system automation: hardware/OS info, Finder reveal/list, file read/write/copy/move/delete, clipboard, notifications, screen capture, app launch/quit, window & process inspection, AppleScript, Shortcuts, and a permissions diagnostic.

ztp macos system-info --json
ztp macos files-write /tmp/out.txt --content "hello" --confirm
ztp macos screenshot-full --output screen.png

🔎 ztp-ocr new in 0.9.0

Apple Vision · PDFKit · on-device · no network
image · pdf · screen · languages

Local, on-device OCR. Recognize text in images, rasterize & OCR scanned PDFs (page ranges + DPI), or capture and read the screen. Returns full text plus per-line confidence and bounding boxes; --languages en,fr constrains recognition and --fast trades accuracy for speed.

ztp ocr image scan.png --json
ztp ocr pdf invoice.pdf --pages 1-2 --dpi 200 --output invoice.txt
ztp ocr screen --json
ztp ocr languages

🗒️ ztp-notes new in 0.9.0

Apple Notes · AppleScript · --confirm gating
list · read · create · append · delete · folders

Read and write Apple Notes, including structured note creation (title becomes the first line; body accepts plain text or HTML). read returns both raw HTML and a plain-text reduction. delete (to Recently Deleted) requires --confirm.

ztp notes create --title "Standup" --body "- shipped OCR\n- next: finder" --json
ztp notes append --name "Standup" --body "- reviewed PRs"
ztp notes read --name "Standup" --json

🗂️ ztp-files new in 0.9.0

FileManager · ditto · no shell · Trash-safe
list · tree · search · info · copy · move · rename · mkdir · delete · compress · extract

Filesystem navigation, search, mutation and (de)compression. search matches by name or, with --content, greps file contents (supports --regex and --ext). delete moves to the Trash by default (--permanent to remove outright). Zip via /usr/bin/ditto — no shell. move/rename/delete/extract require --confirm.

ztp files list --path ~/Documents --sort date --json
ztp files search --path ~/src --query "TODO" --content --ext swift --json
ztp files compress --path ~/Project --output ~/project.zip
ztp files delete --path ~/old.txt --confirm

🪟 ztp-finder new in 0.9.0

Finder · AppleScript · --confirm gating
selection · reveal · open · new-window · set-view · info · trash · empty-trash · eject

Advanced Finder control. selection returns the POSIX paths of the currently selected items; set-view switches the front window between icon / list / column / gallery. trash, empty-trash and eject require --confirm.

ztp finder selection --json
ztp finder set-view --view column --path ~/Downloads
ztp finder reveal --path ~/report.pdf
ztp finder eject --name "Backup" --confirm

Architecture #

ztp (ZTPCLI)
├── ZTPProtocols     ZTPTool, ToolManifest, ToolInput/Result, JSONValue
├── ZTPCore          ToolRegistry, ZTPRuntime (actor), EventBus, Logger
├── ZTPExcel  ZTPDocx  ZTPSlides  ZTPChart      ← document & visual generators
├── ZTPMail   ZTPMessage  ZTPBrowser  ZTPMacOS  ← comms, web & system
└── ZTPOCR    ZTPNotes    ZTPFiles    ZTPFinder ← OCR, Apple Notes, files & Finder

Each generator owns its Schema → Model → OpenXML/render → package pipeline with a dedicated validator. The CLI exposes both per-tool subcommands and the generic protocol commands.

Security model #

  • Confirmation gating — every mutating/destructive operation (send, write, move, delete, trash, eject…) requires an explicit --confirm/--confirmed flag.
  • AppleScript safety — all interpolated values are escaped (quote, backslash, newline, tab) and osascript runs through Process argument arrays, never a shell string.
  • SSRF guard — the browser blocks private, loopback, and cloud-metadata hosts by default.
  • No shell for archives — compression uses /usr/bin/ditto via argument arrays.
  • On-device OCR — Vision runs locally; no image or text leaves the machine.
Confirmation is consent: ZTP does not second-guess a caller who explicitly passes --confirm on a destructive action.
